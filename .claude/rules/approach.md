# Default Approach

- Read existing code before making changes — understand patterns already in use
- Follow existing conventions in the codebase (naming, file structure, component patterns)
- Prefer editing existing files over creating new ones
- Keep changes minimal and focused on what was asked
- Don't over-engineer — solve the current problem, not hypothetical future ones

## Decision Heuristics

- "Is there an existing component/pattern for this?" → use it
- "Am I adding this for a hypothetical future?" → don't
- "Can I fix this in the existing file?" → do that before creating new files
